# Kadane's Algorithm (Maximum Subarray)

> **Tier 0 — Foundations** | Prerequisites: Time & Space Complexity, Prefix Sum Techniques | Required by: Sliding Window (Tier 1), Dynamic Programming (Tier 4)

---

## 1. Intuition First (No Jargon)

**The core idea:** At each position, you make a binary decision: either extend the current subarray by including this element, or start a brand new subarray from this element. The decision is simple — whichever gives a larger sum. Track the global maximum across all these decisions.

**Real-world analogy:** You're reviewing daily P&L for a trading desk. Each day has a profit or loss. You want to find the most profitable consecutive streak. At each day, you ask: "Am I better off continuing my current streak (which might have had some bad days but still nets positive), or cutting my losses and starting fresh from today?" If the accumulated streak is negative, it's dragging you down — start fresh. If it's still positive, keep riding it. That's Kadane's.

**WHY this pattern exists:** Finding the maximum sum contiguous subarray seems like it needs O(n²) to check all subarrays. Kadane's collapses this to O(n) with a key insight: you never need to consider a subarray that starts from a prefix with negative sum, because dropping that prefix always helps. This is the simplest and most elegant example of dynamic programming — a single variable carrying the optimal state forward.

**The mental shift:** You're not searching subarrays. You're making a local decision at each step: extend or restart. The globally optimal subarray emerges from these local decisions. This "locally optimal → globally optimal" property is what makes it work.

## 2. Pattern Recognition

### When to use

| Signal | Example |
|--------|---------|
| "Maximum sum subarray" (no size constraint) | Classic Kadane's |
| "Maximum sum subarray" (with negatives) | Sliding window breaks → Kadane's |
| "Maximum product subarray" | Modified Kadane's tracking min AND max |
| "Maximum circular subarray sum" | Kadane's + total sum trick |
| "Best time to buy and sell stock" | Kadane's on the diff array (daily price changes) |
| Any "contiguous subarray optimization" where the answer can extend or restart | Kadane's family |

### Common keywords in problem statements

- "maximum sum subarray"
- "contiguous", "consecutive"
- "maximum/minimum sum"
- "best", "optimal" streak/range
- Problems that mention arrays with negative numbers and ask for subarray optimization

### 10-Second Identification Rule

> **Is it asking for the maximum (or minimum) sum contiguous subarray, with no constraint on size, and the array has negative numbers?**
>
> If YES → Kadane's. Period. O(n) time, O(1) space.
>
> If there's a size constraint (exactly k, or at most k) → Sliding Window, not Kadane's.
> If you need the count or existence of subarrays with exact sum → Prefix Sum + HashMap.
> If the array is all positive → The whole array is the answer (or use sliding window for a constraint).

### Kadane's vs Sliding Window vs Prefix Sum

| | Kadane's | Sliding Window | Prefix Sum + HashMap |
|---|---|---|---|
| **Problem** | Max sum subarray | Optimal length subarray with constraint | Count/find subarrays with exact sum |
| **Negatives** | Handles them | Breaks (sum-based) | Handles them |
| **Size constraint** | None | Yes | No |
| **Complexity** | O(n) time, O(1) space | O(n) time, O(1) space | O(n) time, O(n) space |
| **Returns** | One optimal value | One optimal length/value | Count or existence |

## 3. Core Algorithm / Template

### Template A: Classic Kadane's (Maximum Subarray Sum)

```csharp
// Returns the maximum sum of any contiguous subarray
// Handles negative numbers, works in O(n) time, O(1) space
public int MaxSubArray(int[] nums)
{
    int maxEndingHere = nums[0];  // best subarray ending at current position
    int maxSoFar = nums[0];      // global best

    for (int i = 1; i < nums.Length; i++)
    {
        // Decision: extend current subarray or start fresh
        maxEndingHere = Math.Max(nums[i], maxEndingHere + nums[i]);
        maxSoFar = Math.Max(maxSoFar, maxEndingHere);
    }

    return maxSoFar;
}
```

**Mental model:** At each index i:
- `maxEndingHere` = max sum subarray that ENDS at position i
- Decision: `max(nums[i], maxEndingHere + nums[i])` = "start fresh" vs "extend"
- If `maxEndingHere` was negative before adding `nums[i]`, starting fresh is better
- `maxSoFar` tracks the global maximum across all positions

### Template B: Kadane's with Subarray Indices

```csharp
// Returns the maximum sum AND the start/end indices of the subarray
public (int sum, int start, int end) MaxSubArrayWithIndices(int[] nums)
{
    int maxEndingHere = nums[0];
    int maxSoFar = nums[0];
    int tempStart = 0;  // potential start of current subarray
    int start = 0, end = 0;

    for (int i = 1; i < nums.Length; i++)
    {
        if (nums[i] > maxEndingHere + nums[i])
        {
            // Starting fresh — new subarray begins at i
            maxEndingHere = nums[i];
            tempStart = i;
        }
        else
        {
            // Extending — keep the current start
            maxEndingHere += nums[i];
        }

        if (maxEndingHere > maxSoFar)
        {
            maxSoFar = maxEndingHere;
            start = tempStart;
            end = i;
        }
    }

    return (maxSoFar, start, end);
}
```

### Template C: Maximum Product Subarray (Modified Kadane's)

```csharp
// Maximum product subarray — track both max AND min
// because a negative × negative can become the largest positive
public int MaxProduct(int[] nums)
{
    int maxProd = nums[0];
    int minProd = nums[0];  // track minimum because negative × negative = positive
    int result = nums[0];

    for (int i = 1; i < nums.Length; i++)
    {
        // If nums[i] is negative, max and min swap roles
        if (nums[i] < 0)
            (maxProd, minProd) = (minProd, maxProd);

        maxProd = Math.Max(nums[i], maxProd * nums[i]);
        minProd = Math.Min(nums[i], minProd * nums[i]);

        result = Math.Max(result, maxProd);
    }

    return result;
}
```

**Key insight for product variant:** Negatives flip max and min. A large negative product × another negative = large positive. So you must track both the maximum AND minimum product ending at each position.

### Template D: Maximum Circular Subarray Sum

```csharp
// Maximum subarray sum in a circular array
// Answer is either:
// (a) normal max subarray (doesn't wrap), OR
// (b) total sum - minimum subarray (wraps around — exclude the min in the middle)
public int MaxSubarraySumCircular(int[] nums)
{
    int totalSum = 0;
    int maxEndingHere = nums[0], maxSoFar = nums[0];
    int minEndingHere = nums[0], minSoFar = nums[0];

    totalSum = nums[0];

    for (int i = 1; i < nums.Length; i++)
    {
        totalSum += nums[i];

        // Standard Kadane's for max
        maxEndingHere = Math.Max(nums[i], maxEndingHere + nums[i]);
        maxSoFar = Math.Max(maxSoFar, maxEndingHere);

        // Kadane's for min (inverted)
        minEndingHere = Math.Min(nums[i], minEndingHere + nums[i]);
        minSoFar = Math.Min(minSoFar, minEndingHere);
    }

    // Edge case: all elements negative → minSoFar = totalSum → circular sum = 0 (invalid)
    if (maxSoFar < 0) return maxSoFar;

    return Math.Max(maxSoFar, totalSum - minSoFar);
}
```

**Why the circular trick works:** A wrapping subarray is everything EXCEPT a non-wrapping subarray in the middle. So max circular sum = totalSum - min non-wrapping subarray. Take the better of wrapping and non-wrapping.

### The DP perspective (what Kadane's really is)

```
Define: dp[i] = maximum sum of subarray ending at index i

Recurrence:
  dp[i] = max(nums[i], dp[i-1] + nums[i])

  Translation: either start fresh at i, or extend from i-1

Answer: max(dp[0], dp[1], ..., dp[n-1])

Space optimization: dp[i] only depends on dp[i-1]
  → Replace array with single variable: maxEndingHere

This IS Kadane's algorithm. It's 1D DP with O(1) space.
```

## 4. Complexity Deep Dive

### Time Complexity

| Variant | Time | Why |
|---------|------|-----|
| Classic Kadane's | O(n) | Single pass, O(1) work per element |
| With indices | O(n) | Same pass, extra bookkeeping |
| Maximum product | O(n) | Same pass, two extra variables |
| Circular | O(n) | Two Kadane's passes (or one combined) |

### Space Complexity

| Variant | Space | Notes |
|---------|-------|-------|
| All variants | O(1) | Only tracking a few variables |
| DP formulation before optimization | O(n) | The dp array |
| After space optimization | O(1) | dp[i] only needs dp[i-1] |

**This is the gold standard:** O(n) time, O(1) space, for a problem that appears to need O(n²).

### Historical context and trade-offs

| Approach | Time | Space | Notes |
|----------|------|-------|-------|
| Brute force (all subarrays) | O(n²) or O(n³) | O(1) | Check all subarrays |
| Prefix sum optimization | O(n²) | O(n) | Range sums in O(1) but still enumerate |
| Divide and conquer | O(n log n) | O(log n) | Split, solve halves, combine crossing |
| Kadane's | O(n) | O(1) | Optimal — can't do better (must read input) |

### When this approach breaks

- **Size constraint:** "Max sum subarray of exactly size k" → Kadane's doesn't restrict size. Use sliding window.
- **Exact target sum:** "Subarray with sum exactly k" → Kadane's finds the max, not a specific target. Use prefix sum + HashMap.
- **Multiple disjoint subarrays:** "Pick two non-overlapping subarrays maximizing total sum" → Kadane's finds one. Need prefix/suffix Kadane's arrays.
- **All positives:** The answer is trivially the entire array. Kadane's still works but it's overkill.

## 5. Edge Cases & Pitfalls

### Common senior-level mistakes

1. **Initializing maxEndingHere = 0 instead of nums[0].** If all elements are negative, `maxEndingHere = 0` gives the wrong answer (0, from an empty subarray). The problem asks for the maximum of NON-EMPTY subarrays. Initialize with `nums[0]` and start the loop at index 1.

2. **Initializing maxSoFar = 0.** Same bug. If all elements are negative, `maxSoFar = 0` is wrong. Initialize with `nums[0]`.

3. **Confusing Kadane's with sliding window.** "Maximum sum subarray of size k" is NOT Kadane's. Kadane's has no size constraint. The sliding window template handles fixed-size windows.

4. **Forgetting the edge case in circular variant.** When ALL elements are negative, `totalSum - minSoFar = 0` (the "wrapping subarray" is empty). But empty subarrays aren't allowed. The check `if (maxSoFar < 0) return maxSoFar;` handles this.

5. **Not tracking both max and min for product variant.** The product version MUST track the minimum product because `negative × negative = positive`. Forgetting the min variable makes the algorithm fail on inputs like `[-2, 3, -4]` (answer: 24 = -2 × 3 × -4).

### Boundary conditions

- Single element → return that element
- All negative → return the largest (least negative) element
- All positive → return the sum of the entire array
- Contains zero → for product variant, zeros reset both max and min to 0
- Very large values → check for integer overflow in sum/product

### Debugging heuristics

1. Print `maxEndingHere` and `maxSoFar` at each step
2. The "restart" decision: `maxEndingHere` should jump to `nums[i]` when `maxEndingHere + nums[i] < nums[i]`, i.e., when `maxEndingHere < 0`
3. For product: check that negative inputs cause proper max/min swaps

## 6. Variations (Pattern Stretching)

### Easy: Classic maximum subarray sum

**Maximum Subarray (LC 53).** Pure Template A. Single pass, track running maximum.

### Medium: Maximum subarray with indices

Return the actual subarray, not just the sum. Template B — track start/end indices alongside the max.

### Medium: Best time to buy and sell stock

**Best Time to Buy and Sell Stock (LC 121).** Equivalent to Kadane's on the *diff array* (daily price changes). Or simpler: track the minimum price seen so far, and the maximum profit from selling at today's price.

```csharp
public int MaxProfit(int[] prices)
{
    int minPrice = prices[0];
    int maxProfit = 0;
    for (int i = 1; i < prices.Length; i++)
    {
        maxProfit = Math.Max(maxProfit, prices[i] - minPrice);
        minPrice = Math.Min(minPrice, prices[i]);
    }
    return maxProfit;
}
```

### Medium-Hard: Maximum product subarray

**Maximum Product Subarray (LC 152).** Template C. Track both max and min products because negatives flip the ordering. Zeros reset both.

### Hard: Maximum circular subarray sum

**Maximum Sum Circular Subarray (LC 918).** Template D. The non-wrapping case is standard Kadane's. The wrapping case: max circular = totalSum - minSubarraySum. Take the better of both. Handle all-negatives edge case.

### Hard: Maximum sum of two non-overlapping subarrays

**Maximum Sum of Two Non-Overlapping Subarrays (LC 1031).** Precompute prefix Kadane's (best subarray in prefix [0..i]) and suffix Kadane's (best subarray in suffix [i..n-1]). For each split point, combine.

### Evolution map

```
Classic max subarray → With indices → Buy/Sell stock (min tracking variant)
         ↓                                      ↓
  Max product (track min too)           Multiple transactions (greedy/DP)
         ↓
  Circular (total - min trick)
         ↓
  Two disjoint subarrays (prefix + suffix Kadane's)
```

## 7. Interview-Level Problems

### Problem 1 (Medium): Maximum Subarray (LC 53)

**Summary:** Given an integer array `nums`, find the subarray with the largest sum and return its sum.

**Expected approach:** Kadane's algorithm. At each position, decide: extend current subarray or start fresh. Track global maximum.

**Key insight:** The decision `maxEndingHere = max(nums[i], maxEndingHere + nums[i])` is equivalent to: "if the best subarray ending at i-1 had negative sum, drop it and start over." This is the core of Kadane's. In interviews, explicitly state this as a DP formulation: "dp[i] = max sum subarray ending at i. Recurrence: dp[i] = max(nums[i], dp[i-1] + nums[i]). Space-optimized to O(1)."

---

### Problem 2 (Medium-Hard): Maximum Product Subarray (LC 152)

**Summary:** Given an integer array `nums`, find the contiguous subarray that has the largest product, and return the product.

**Expected approach:** Modified Kadane's tracking both maximum and minimum product at each position. Before updating, check if `nums[i]` is negative — if so, swap max and min (because multiplying a negative by the min gives the max).

**Key insight:** The product variant breaks the standard Kadane's invariant because negatives flip signs. The fix — tracking both max AND min — is a pattern that appears in any "optimize over a function with sign flips." Zeros reset everything to 0 (or 1, depending on how you frame it). Interviewers specifically test whether you remember to track the minimum.

---

### Problem 3 (Hard): Maximum Sum Circular Subarray (LC 918)

**Summary:** Given a circular integer array `nums`, find the maximum possible sum of a non-empty subarray.

**Expected approach:** Run standard Kadane's for max subarray (non-wrapping case). Run inverted Kadane's for min subarray (wrapping case: max_circular = totalSum - minSubarray). Answer is `max(maxKadane, totalSum - minKadane)`. Special case: if all elements are negative, the circular case gives 0 (empty subarray), which is invalid → return maxKadane.

**Key insight:** The circular decomposition — a wrapping subarray's complement is a non-wrapping subarray — is the non-obvious insight. Once you see it, the problem reduces to two Kadane's runs. The all-negatives edge case is the trap: `totalSum - minSoFar = 0`, but empty subarrays aren't allowed. Interviewers test exactly this case.

## 8. Staff-Level Thinking

### Trade-offs between approaches

| Approach | When to use | Why not always |
|----------|------------|---------------|
| Kadane's | Max/min sum subarray, no constraints | Only finds one optimal value; can't count, can't target specific sum |
| Prefix Sum + HashMap | Count subarrays with exact sum k | O(n) space; more complex to implement |
| Sliding Window | Min/max length with sum constraint, positives only | Breaks with negatives |
| Divide & Conquer | Educational / follow-up question | O(n log n) — strictly worse than Kadane's |
| Brute Force | n ≤ 100, as a verification tool | O(n²) or O(n³) — doesn't scale |

### Scalability considerations

- Kadane's is the theoretical lower bound for this problem — O(n) time, O(1) space. You can't do better because you must read every element at least once.
- In streaming contexts, Kadane's works on unbounded streams: process each element as it arrives, never needing to revisit. This makes it suitable for real-time analytics on data pipelines.
- For the "sliding window maximum" variant (max subarray of exactly size k), Kadane's doesn't apply — but the sliding window version is also O(n). Different tool, same complexity.

### When NOT to use Kadane's

- **You need all subarrays with a property, not just the best one.** Kadane's finds THE max. For counting or enumerating, you need prefix sum or DP.
- **There's a size constraint.** Kadane's doesn't restrict subarray size. Fixed-size → sliding window. Variable size with constraint → sliding window or binary search.
- **You need the second-best, or top-K subarrays.** Kadane's only tracks the global optimum. Extending to top-K requires fundamentally different approaches.
- **Multi-dimensional arrays.** Kadane's is 1D. For 2D max subarray sum, you fix two row boundaries (O(n)) and apply Kadane's to column sums → O(n²m) or O(nm²).

### How to explain this in interviews

**Opening (10 seconds):** "This is the maximum subarray problem — solved optimally by Kadane's algorithm. O(n) time, O(1) space."

**DP framing (15 seconds):** "I'll think of it as DP where `dp[i]` is the max sum subarray ending at index i. The recurrence is `dp[i] = max(nums[i], dp[i-1] + nums[i])`. Since dp[i] only depends on dp[i-1], I can space-optimize to a single variable."

**Decision explanation (10 seconds):** "At each position, I either extend the current subarray or start a new one. I extend if the running sum is non-negative — it's worth carrying forward. I restart if the running sum is negative — it would drag down anything I attach to it."

## 9. Real-World Mapping

| Production scenario | Kadane's analog |
|---|---|
| **Trading P&L analysis** | Maximum profit streak = max subarray sum on daily P&L. Used in quantitative finance for strategy evaluation. |
| **User engagement** | Longest "hot streak" of positive session deltas. If engagement dips, start counting from the next positive day. |
| **Network throughput** | Find the time window with maximum data transferred. Kadane's on throughput samples with overhead (negative) periods. |
| **Load testing analysis** | Find the burst window where the system handled maximum aggregate load. Each sample = requests processed - requests dropped. |
| **eSports tournament scoring** | Maximum scoring streak for a player across rounds (some rounds may have negative point swings). |
| **Marketing campaign ROI** | Find the best consecutive period of campaign performance. Daily ROI can be negative; Kadane's finds the best run. |

**System design relevance:** Kadane's itself rarely appears directly in system design. But the PRINCIPLE — "maintain a running optimal state, make local decisions that compose into a global optimum" — is the foundation of greedy algorithms, which appear everywhere in scheduling, resource allocation, and optimization. Understanding Kadane's as the simplest DP → greedy bridge is the deeper takeaway.

## 10. 10-Second Recall Cheat Sheet

```
CORE ALGORITHM:
  maxEndingHere = max(nums[i], maxEndingHere + nums[i])
  maxSoFar = max(maxSoFar, maxEndingHere)
  Translation: extend or restart, track global best

INITIALIZATION (CRITICAL):
  maxEndingHere = nums[0]  (NOT 0)
  maxSoFar = nums[0]       (NOT 0)
  Loop from i = 1           (NOT i = 0)

VARIANTS:
  Max sum subarray           → Classic Kadane's
  Max product subarray       → Track BOTH max AND min (negatives flip)
  Circular max subarray      → max(Kadane_max, totalSum - Kadane_min)
  Buy/Sell stock             → Track minPrice, compute maxProfit at each step
  Two non-overlapping        → Prefix Kadane's + Suffix Kadane's

KADANE'S vs SLIDING WINDOW:
  No size constraint + negatives → Kadane's
  Size constraint (fixed k)     → Sliding Window
  Length optimization + positives → Sliding Window

DP FORMULATION:
  dp[i] = max(nums[i], dp[i-1] + nums[i])
  answer = max(dp[0..n-1])
  Space: dp[i] only needs dp[i-1] → O(1)

PITFALLS:
  - Init maxSoFar = 0 → wrong when all negative
  - Confusing with sliding window (Kadane's has NO size constraint)
  - Product variant: must track min product too
  - Circular variant: all negatives → totalSum - minSoFar = 0 → return maxSoFar instead
```
