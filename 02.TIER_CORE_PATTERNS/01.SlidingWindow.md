# Sliding Window (Fixed & Variable)

> **Tier 1 — Core Patterns** | Prerequisites: Tier 0 (Prefix Sum, Kadane's, Complexity Analysis) | Adjacent: Two Pointers, Hashing

---

## 1. Intuition First (No Jargon)

**The core idea:** You have a contiguous subarray/substring you care about. Instead of recomputing everything from scratch when you shift your focus by one position, you *incrementally update* — drop the element leaving the window, add the element entering.

**Real-world analogy:** You're monitoring a rolling 5-minute average of request latency on a Grafana dashboard. Every second, you don't re-sum the last 300 data points. You subtract the oldest sample that just fell out of the window and add the newest one. That's a fixed sliding window. Now imagine your alerting rule says "find the shortest time span where p99 exceeds 500ms." The window size isn't fixed — it *grows* until the condition is met, then *shrinks* from the left to find the minimum span. That's a variable (shrinking) window.

**WHY this pattern exists:** Brute force over all subarrays is O(n²) or O(n³). Sliding window exploits the *overlap* between consecutive windows to collapse this to O(n). It's the canonical technique for **contiguous subarray/substring optimization** problems.

**Key mental shift:** You're not iterating over subarrays. You're maintaining a *state* (sum, frequency map, count of valid characters) that you update in O(1) as the window slides.

## 2. Pattern Recognition

### When to use

| Signal | Example |
|--------|---------|
| "contiguous subarray/substring" | Maximum sum subarray of size k |
| "at most K distinct" | Longest substring with at most K distinct characters |
| "minimum length subarray with sum >= target" | Variable window — shrink when condition met |
| "all anagrams / permutation in string" | Fixed window = length of pattern |
| "maximum/minimum over all windows of size k" | Fixed window (often combined with monotonic deque) |

### Common keywords in problem statements

- "subarray", "substring", "contiguous"
- "maximum", "minimum", "longest", "shortest"
- "at most K", "exactly K", "no more than"
- "sum >= target", "contains all characters"
- "consecutive", "window", "sliding"

### 10-Second Identification Rule

> **Is the problem asking for an optimal (max/min/count) contiguous subarray or substring, and can you maintain the answer state incrementally as endpoints move right?**
>
> If YES → Sliding Window.
> If the array has negative numbers and you need max subarray sum → Kadane's (not sliding window).
> If it's about subsequences (not contiguous) → DP or Two Pointers, not sliding window.

### Two flavors — pick the right one immediately

| | Fixed Window | Variable Window |
|---|---|---|
| **Window size** | Known upfront (k given) | Unknown — you're optimizing for it |
| **Movement** | Both pointers advance together | Right expands, left shrinks conditionally |
| **Template** | Simpler: slide by 1 each step | Needs inner while loop for shrinking |
| **Example** | Max sum of subarray of size k | Shortest subarray with sum >= target |

## 3. Core Algorithm / Template

### Template A: Fixed Window

```csharp
// Fixed-size sliding window
// Returns the maximum sum of any subarray of exactly size k
public int MaxSumFixedWindow(int[] nums, int k)
{
    int n = nums.Length;
    if (n < k) return -1; // or throw

    int windowSum = 0;

    // Build the first window
    for (int i = 0; i < k; i++)
        windowSum += nums[i];

    int maxSum = windowSum;

    // Slide: add right edge, remove left edge
    for (int i = k; i < n; i++)
    {
        windowSum += nums[i];       // expand right
        windowSum -= nums[i - k];   // shrink left
        maxSum = Math.Max(maxSum, windowSum);
    }

    return maxSum;
}
```

**Mental model:** Build first window → slide one step at a time → update state with +1 / -1 element.

### Template B: Variable Window (Shrinking / Minimum)

```csharp
// Variable-size sliding window
// Returns the minimum length subarray with sum >= target
// Returns 0 if no such subarray exists
public int MinSubarrayLen(int target, int[] nums)
{
    int n = nums.Length;
    int left = 0;
    int windowSum = 0;
    int minLen = int.MaxValue;

    for (int right = 0; right < n; right++)
    {
        windowSum += nums[right]; // expand

        // Shrink from left while condition holds
        while (windowSum >= target)
        {
            minLen = Math.Min(minLen, right - left + 1);
            windowSum -= nums[left];
            left++;
        }
    }

    return minLen == int.MaxValue ? 0 : minLen;
}
```

**Mental model:** Right pointer always moves forward (expand). Left pointer moves forward only when the window satisfies the condition (shrink to optimize). Each element enters and leaves the window at most once → O(n).

### Template C: Variable Window with Frequency Map (String Problems)

```csharp
// Longest substring with at most K distinct characters
public int LongestSubstringKDistinct(string s, int k)
{
    int n = s.Length;
    if (n == 0 || k == 0) return 0;

    var freq = new Dictionary<char, int>();
    int left = 0;
    int maxLen = 0;

    for (int right = 0; right < n; right++)
    {
        char c = s[right];
        freq[c] = freq.GetValueOrDefault(c, 0) + 1;

        // Shrink until we have at most k distinct
        while (freq.Count > k)
        {
            char leftChar = s[left];
            freq[leftChar]--;
            if (freq[leftChar] == 0)
                freq.Remove(leftChar);
            left++;
        }

        maxLen = Math.Max(maxLen, right - left + 1);
    }

    return maxLen;
}
```

**Reusable mental model for all three:**

```
Initialize: left = 0, state = empty
For each right in [0, n):
    1. ADD nums[right] / s[right] to state
    2. SHRINK from left while window is INVALID (or while condition is MET for minimization)
    3. UPDATE answer
```

## 4. Complexity Deep Dive

### Time Complexity

| Variant | Time | Why |
|---------|------|-----|
| Fixed window | O(n) | Single pass, O(1) work per step |
| Variable window | O(n) | Each element enters window once, leaves once → at most 2n pointer moves |
| Variable + frequency map | O(n) | Same reasoning; dictionary ops are O(1) amortized |

**The non-obvious part:** The variable window looks like O(n²) because of the nested while loop. It's not. The `left` pointer only moves right, never resets. Total left pointer movements across all iterations of the outer loop ≤ n. So total work = O(n) for right + O(n) for left = O(n).

### Space Complexity

| Variant | Space | Notes |
|---------|-------|-------|
| Sum-based | O(1) | Just tracking sum and pointers |
| Frequency map | O(min(n, alphabet)) | Bounded by character set (26 for lowercase) or k |
| With auxiliary structure (monotonic deque) | O(k) | For sliding window maximum/minimum |

### When this approach breaks

- **Negative numbers + sum target:** You can't shrink from the left when the window sum exceeds the target because removing a negative number *increases* the sum. Use prefix sum + sorted structure or Kadane's instead.
- **Non-contiguous selections:** Sliding window requires contiguity. Subsequence problems need DP.
- **Multiple disjoint windows:** If you need to pick 2 non-overlapping subarrays, sliding window alone won't work — combine with prefix/suffix arrays.

## 5. Edge Cases & Pitfalls

### Common senior-level mistakes

1. **Off-by-one on window boundaries.** The window `[left, right]` has size `right - left + 1`. Forgetting the `+1` is the #1 bug. Draw it out: `left=2, right=4` → elements at indices 2, 3, 4 → size 3, not 2.

2. **Not handling "exactly K" correctly.** "Exactly K distinct" ≠ "at most K distinct." The trick:
   ```
   exactlyK(k) = atMostK(k) - atMostK(k - 1)
   ```
   This is a critical identity. Memorize it.

3. **Shrinking too early or too late.** For *maximization* problems (longest substring), shrink only when the window becomes *invalid*. For *minimization* problems (shortest subarray), shrink while the window is *valid*. Mixing these up gives wrong answers silently — tests may even pass on small inputs.

4. **Resetting left pointer.** Left pointer NEVER resets to 0 or to right. It only moves forward. If you find yourself resetting, you've broken the invariant.

5. **Using sliding window when Kadane's is correct.** "Maximum subarray sum" (no size constraint, has negatives) → Kadane's. "Maximum subarray sum of size k" → sliding window. The distinction matters.

### Boundary conditions

- Empty array / string → return 0 or handle explicitly
- `k > n` → no valid window exists
- `k = 0` → degenerate case, return 0
- All elements identical → window may span entire array
- Single element array → window is the element itself

### Debugging heuristic

If your sliding window gives wrong answers:
1. Print `left`, `right`, and window state at each step
2. Check: does `left` ever decrease? (Bug if yes)
3. Check: is the shrink condition inverted? (maximize vs minimize)
4. Check: are you updating the answer at the right moment? (After shrink for minimization, inside the valid window for maximization)

## 6. Variations (Pattern Stretching)

### Easy: Fixed window, simple state

**Max sum subarray of size k** — Pure template A. Build first window, slide.

### Medium: Variable window, single constraint

**Minimum size subarray sum ≥ target** (positive integers) — Template B. Expand right, shrink left while valid, track minimum length.

**Longest substring without repeating characters** — Template C with constraint: frequency of every character ≤ 1. Shrink when a duplicate enters.

### Medium-Hard: Variable window, compound state

**Longest substring with at most K distinct characters** — Template C. Frequency map tracks distinct count. Shrink when `freq.Count > k`.

**Minimum window substring (contain all characters of T in S)** — Two frequency maps: one for target, one for window. Track `formed` count (how many characters in target are satisfied). Shrink while fully formed.

### Hard: Sliding window + auxiliary data structure

**Sliding window maximum** — Fixed window but you need max in O(1). Combine with monotonic deque (Tier 3 topic). The deque maintains decreasing order; front is always the max.

**Subarrays with K different integers (exactly K)** — Apply the `exactlyK = atMostK(k) - atMostK(k-1)` transform. Run variable window twice.

**Longest substring with at most K replacements** — Track frequency of the most common character in the window. Window is valid if `windowSize - maxFreq <= k`. Shrink when invalid.

### Evolution map

```
Fixed Sum → Variable Sum → Variable with Map → Variable with "Exactly K" trick
     ↓                                              ↓
  + Monotonic Deque                        + Two-map matching
  (Sliding Window Max)                  (Minimum Window Substring)
```

## 7. Interview-Level Problems

### Problem 1 (Medium): Longest Substring Without Repeating Characters

**LC 3**

**Summary:** Given a string `s`, find the length of the longest substring without repeating characters.

**Expected approach:** Variable sliding window with a HashSet or frequency array. Expand right. When a duplicate is found, shrink from left until the duplicate is removed.

**Key insight:** You can optimize the shrink step: instead of incrementally moving left, store the *last seen index* of each character and jump left directly to `lastSeen[c] + 1`. This avoids redundant left-pointer steps but doesn't change O(n) complexity — it reduces constant factor.

---

### Problem 2 (Medium-Hard): Minimum Window Substring

**LC 76**

**Summary:** Given strings `s` and `t`, find the minimum window in `s` that contains all characters of `t` (including duplicates).

**Expected approach:** Two frequency maps — `need` (from `t`) and `have` (from current window). Maintain a `formed` counter tracking how many unique characters in `t` are fully satisfied in the window. Expand right to grow. Shrink left while `formed == required` to minimize.

**Key insight:** The `formed` counter avoids comparing full maps on every step. Only increment `formed` when `have[c]` reaches `need[c]`, only decrement when it drops below. This keeps the inner loop O(1) per character operation.

---

### Problem 3 (Hard): Subarrays with K Different Integers

**LC 992**

**Summary:** Given an integer array `nums` and integer `k`, return the number of subarrays with exactly `k` distinct integers.

**Expected approach:** Use the identity `exactly(k) = atMost(k) - atMost(k-1)`. Implement `atMost(k)` as a variable sliding window that counts ALL valid subarrays (not just the longest). For each right position, once the window `[left, right]` is valid (≤ k distinct), all subarrays `[left, right], [left+1, right], ..., [right, right]` are valid → add `right - left + 1` to count.

**Key insight:** The counting step `right - left + 1` is the non-obvious part. Once you internalize that for a valid window ending at `right`, the number of valid subarrays ending at `right` equals the window size — the whole pattern clicks. The `exactly = atMost - atMost` transform is reusable across many "exactly K" problems.

## 8. Staff-Level Thinking

### Trade-offs between approaches

| Approach | When to prefer | Downside |
|----------|---------------|----------|
| Sliding Window | Contiguous, monotonic constraint, O(n) needed | Can't handle negatives with sum constraints |
| Prefix Sum + Binary Search | Contiguous, has negatives, sum constraint | O(n log n) — slower but more general |
| Prefix Sum + HashMap | Exact sum target with negatives | O(n) but uses O(n) space |
| Kadane's | Max subarray sum, no size constraint | Only solves one specific problem |

### Scalability considerations

- Sliding window is O(n) time, O(1) or O(k) space. It's the *best you can do* for contiguous subarray problems — there's no asymptotic improvement possible since you must read every element.
- In production streaming contexts (Kafka consumers, metrics pipelines), the sliding window pattern maps directly to **tumbling windows** and **hopping windows** in stream processing frameworks (Flink, Kafka Streams). The mental model is identical.
- For very large datasets that don't fit in memory, the fixed-size sliding window is naturally streamable — you only need O(k) memory. Variable windows can require O(n) in the worst case (window spans entire input).

### When NOT to use this pattern

- Array has negative numbers and you need minimum/maximum sum subarray of variable size → Kadane's or prefix sum.
- The selection doesn't need to be contiguous → This is DP/greedy territory.
- You need to find two or more disjoint subarrays → Sliding window alone won't work.
- The "window validity" condition isn't monotonic (shrinking the window can make it both more valid and less valid unpredictably) → Sliding window invariant breaks.

### How to explain this in interviews

**Opening (10 seconds):** "This is a sliding window problem because we need an optimal contiguous subarray/substring, and the window validity is monotonic — expanding can only add elements, shrinking can only remove them."

**Template articulation (20 seconds):** "I'll maintain two pointers, left and right. Right always advances. Left advances when the window needs to shrink — either because it exceeded the size constraint or because the condition is already satisfied and I'm optimizing. Each element enters and exits the window exactly once, so this is O(n)."

**Why O(n) — the question they WILL ask:** "The nested while loop looks like O(n²), but the left pointer is monotonically increasing across all iterations of the outer loop. Total left-pointer movements ≤ n. So it's O(n + n) = O(n), not O(n × n)."

## 9. Real-World Mapping

| Production scenario | Sliding window analog |
|---|---|
| **Rolling metrics** (p99 latency over last 5 min) | Fixed window over time-series data |
| **Rate limiting** (max 100 requests per 60-second window) | Fixed window with count state; token bucket is the production-grade version |
| **Stream processing** (Kafka Streams windowed aggregations) | Tumbling = non-overlapping fixed windows; Hopping = overlapping fixed windows; Session = variable windows |
| **Network congestion control** (TCP sliding window) | Variable window — grows when ACKs arrive, shrinks on packet loss |
| **Log analysis** ("find the shortest time range containing all error types") | Variable window with frequency map — identical to Minimum Window Substring |
| **Anomaly detection** ("alert if error rate exceeds threshold in any K-second span") | Fixed window scan over event stream |

**Backend design relevance:** When you're designing a rate limiter in a system design interview, explicitly call out: "This is fundamentally a fixed sliding window. The production concern is whether we use a true sliding window (more accurate, more state) or a fixed-window counter (simpler, allows bursts at boundaries). The sliding window log approach stores each request timestamp; the sliding window counter interpolates between the current and previous fixed windows."

## 10. 10-Second Recall Cheat Sheet

```
TRIGGER → PATTERN:
  "contiguous subarray/substring" + "optimal"     → Sliding Window
  "size k" or "fixed length"                       → Fixed Window (Template A)
  "minimum length", "shortest"                     → Variable Window, shrink while VALID
  "maximum length", "longest"                      → Variable Window, shrink while INVALID
  "at most K distinct"                             → Variable + Frequency Map
  "exactly K distinct"                             → atMost(K) - atMost(K-1)
  "contains all characters of T"                   → Variable + Two Maps + formed counter
  "max/min in every window of size k"              → Fixed Window + Monotonic Deque

INVARIANTS:
  - left never decreases
  - right always moves forward by 1
  - each element enters and exits at most once → O(n)

PITFALLS:
  - Window size = right - left + 1 (not right - left)
  - Negatives break sum-based variable windows
  - Maximize → shrink when INVALID | Minimize → shrink while VALID
  - "Exactly K" = atMost(K) - atMost(K-1)
```
